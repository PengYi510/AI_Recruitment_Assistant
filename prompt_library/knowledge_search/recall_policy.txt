# 目标
这是一个根据用户query，进行知识库召回的任务。

知识库包含知识类别和文本两个字段，具体如下：
1）know_type：知识类别，取值如下：
- 公司名称：1）公司的标准化名称和各类公司主体的名称。例如：美团：北京三快在线科技有限公司，阿里巴巴：阿里、饿了么、蚂蚁金服、菜鸟网络。2）一类公司的统称，例如：三中一华、MBB、四大、互联网大厂、ATM、BAT等。
- HR专业名词：HR领域的行业名词，例如：CoE、HRBA、HRBP、SSC、TD等。
- 考试或证书：常用的考试或证书名称，例如：全国大学生英语等级考试、专业英语等级考试、会计从业资格证、教师资格证、驾驶证等。
- 美团内部技术：美团公司内部的技术或平台，例如：Friday模型工厂、CatPaw、万象、AIBD等。
- 美团内部业务或部门：1）美团业务，例如：小象超时、快驴、歪马送酒、松鼠便利、美团无人机、易生活、浣熊食堂等。2）美团内部部门或组织简称，例如：CAP部门、企平部门等。
2）text：知识库文本。

# 任务要求
1.首先判断使用哪种知识类别（即know_type的取值）做召回，然后判断输入给该知识类别做召回的关键词有哪些。关键词不仅局限于用户query中的关键词，你也可以自己做扩展，目标是保证能召回合适的知识。know_type必须是以上取值中的某一个，不要输出其他值，也不要对取值名称做任何修改。
2.结果用数组返回，如果你认为用户query需要多个知识类别做召回，那么返回全部结果。务必保证每个知识类别只返回一条记录，多个召回关键词用数组格式返回。
3.如果没有合适的知识类别做召回，或者你无法判断，返回空数组[]。
4.严格按照以下JSON格式输出结果，确保能被python程序正确解析。
```json
[
  {{"know_type": "", "recall_keywords": ["", ""]}}
]
```

# 约束条件
- 不要用职级信息做召回，比如：L5、L7、O4、O5、P8等
- 不要用序列做召回，比如：商分（商业分析）、商品采购、物流管理

# 输入用户query
```
{query}
```
